## Responsabilità

`app/settings.py` è la **Single Source of Truth** per tutti i parametri configurabili. Non contiene logica: solo costanti. È l'unico concern trasversale che tutti i livelli possono leggere.

**Livello**: Infrastruttura (letto da tutti i livelli; scritto solo dalla calibrazione)

<Warning>
`settings.py` viene modificato **a runtime** dal sistema di calibrazione tramite sostituzione regex. Non aggiungere logica Python che dipenda dall'ordine di valutazione delle variabili in questo file.
</Warning>

## Posizioni ROI — Riferimento Completo

Tutte le coordinate sono in pixel relativi all'angolo in alto a sinistra della finestra PokerTH (1280×720).

### Carte Hero — `HERO_CARD_POSITIONS`

Lista di 2 elementi `[x, y]` — angolo superiore sinistro di ogni carta.

| Indice | Carta |
|---|---|
| `[0]` | Prima carta hero |
| `[1]` | Seconda carta hero |

### Carte Board — `BOARD_CARD_POSITIONS`

Lista di 5 elementi `[x, y]`.

| Indice | Carta |
|---|---|
| `[0]–[2]` | Flop (1ª, 2ª, 3ª carta) |
| `[3]` | Turn |
| `[4]` | River |

### Posizioni Dealer Button — `BUTTON_POSITIONS`

Lista di 6 elementi `[x, y, w, h]` — una ROI per ogni seat (0=Hero, 1–5=Avversari).

### Posizioni Bet — `BET_POSITIONS`

Lista di 6 elementi `[x, y, w, h]`. OCR Tesseract viene applicato a ciascuna ROI bet.

### Posizioni Giocatori — `PLAYER_POSITIONS`

Lista di 6 elementi `[x, y, w, h]`. Usate per rilevare la presenza di un giocatore attivo (card-back detection HSV).

### Piatto — `POT_AREA`

```python
POT_AREA = [457, 365, 105, 20]   # [x, y, w, h]
```

## Soglie e Parametri di Rilevamento

| Parametro | Default | Descrizione |
|---|---|---|
| `POT_RESET_THRESHOLD` | `5.0` | Pot minimo (€/chips) che deve essere stato raggiunto per considerare valida una transizione a nuova mano |
| `COLOR_MIN_PIXELS` | `30` | Pixel HSV minimi per confermare un seme colorato (cuori, quadri, fiori) |
| `COLOR_BLACK_MIN_PIXELS` | `20` | Soglia per le picche (bassa saturazione su sfondo verde) |
| `PLAYER_CARD_BACK_MIN_PCT` | `10.0` | Percentuale minima pixel card-back per rilevare un avversario attivo |
| `DEALER_WHITE_MIN_PCT` | `50.0` | Percentuale minima pixel bianchi per rilevare il dealer button |
| `MAX_INVALID_READS` | `3` | Letture invalide consecutive tollerabili prima di accettare un'animazione come stato reale |
