## `Decision` вЂ” Enum delle Azioni

| Valore | Stringa | Significato |
|---|---|---|
| `RAISE` | `"RAISE"` | Rilancia |
| `CALL` | `"CALL"` | Chiama |
| `FOLD_CHECK` | `"FOLD/CHECK"` | Passa/Folda |
| `FOLDED` | `"FOLDED"` | L'hero ha giГ  folded |
| `WAIT` | `"Aspetta il prossimo Preflop"` | Dati insufficienti o fase non preflop |

## `PreflopDecisionEngine`

### `evaluate(snapshot) в†’ Decision`

Catena di guardie in ordine di prioritГ :

```python
# 1. Non siamo al Preflop?
if snapshot["game_state"] != "Preflop":
    return Decision.WAIT

# 2. Sistema non ancora calibrato?
if not history.is_calibrated():
    return Decision.WAIT

# 3. Lettura compromessa in questa mano?
if history.is_hand_compromised():
    return Decision.COMPROMISED

# 4. L'hero non ha 2 carte?
if len(snapshot["player_cards"]) < 2:
    return Decision.WAIT

# 5. Hand key non valida?
key = hand_key(card1, card2)
if key == "":
    return Decision.WAIT

# 6. Posizione sconosciuta?
if snapshot["relative_position"] == "Unknown":
    return Decision.WAIT

# 7. Lookup nella tabella corretta
n_active = history.n_players_at_start()
hero_pos = normalize_position(snapshot["relative_position"], n_active)

if history.is_rfi():
    if hero_pos == "BB":
        return Decision.FOLD_CHECK   # BB in RFI в†’ check/fold gratuito
    table = RFI_TABLES.get(hero_pos)
else:
    raiser_pos = history.raiser_position()
    table = VS_RAISE_TABLES.get(f"VS_{raiser_pos}")

action = lookup(table, key)   # "raise" / "call" / "fold"
return _ACTION_TO_DECISION[action]
```

### `_ACTION_TO_DECISION`

```python
{
    "raise": Decision.RAISE,
    "call":  Decision.CALL,
    "fold":  Decision.FOLD_CHECK
}
```

## Caso Speciale: BB in RFI

Quando l'hero ГЁ in BB e non c'ГЁ stato nessun raise (`is_rfi() == True`), la mano ГЁ passata a lui con la possibilitГ  di check gratuito вЂ” non c'ГЁ nessun entry in `RFI_TABLES` per BB. Il risultato ГЁ sempre `Decision.FOLD_CHECK` (=check nel contesto BB).
