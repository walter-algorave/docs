## Responsabilità

`app/calibration_params.py` definisce le **strutture dati e funzioni** che il sistema di calibrazione usa per gestire i 27 parametri calibrabili, salvare i valori calibrati in `calibration_data.json` e mantenere baseline/valori correnti.

**Livello**: Presentazione (supporto a `CalibrationWindow`)

---

## `_Ref` — Contenitore Valore Mutabile

```python
class _Ref:
    def __init__(self, value):
        self.value   = value    # valore corrente (aggiornato dalla calibrazione)
        self._origin = value    # baseline (valore alla creazione della sessione)

    def restore(self):
        self.value = self._origin    # ripristina alla baseline

    def reset(self, new_value):
        self.value   = new_value     # aggiorna sia corrente che baseline
        self._origin = new_value     # (dopo un salvataggio)
```

## `_load_settings_values() → dict`

Legge i valori attuali da `settings` e li restituisce come dict piatto con chiavi nominative:

```python
{
    "POT_AREA":     [457, 365, 105, 20],
    "HERO_CARD_0":  [x, y],
    "HERO_CARD_1":  [x, y],
    "BOARD_CARD_0": [x, y],
    ...
    "BUTTON_0":     [x, y, w, h],
    "BUTTON_1":     [x, y, w, h],
    ...
    "BET_0":        [x, y, w, h],
    ...
    "PLAYER_0":     [x, y, w, h],
    ...
}
```

Totale: 1 (POT) + 2 (HERO) + 5 (BOARD) + 6 (BUTTON) + 6 (BET) + 6 (PLAYER) = **26 parametri** distinti (27 entry nel dropdown contado con POT_AREA).

---

## `build_refs() → dict[str, _Ref]`

Chiama `_load_settings_values()` e costruisce un `_Ref` per ogni parametro. Il risultato è il dizionario `refs` passato a `build_params()`.

---

## `build_params(refs) → list[dict]`

Costruisce la lista completa dei **27 parametri** usati dal dropdown di `CalibrationWindow`. Ogni parametro è un dict con:

| Chiave | Tipo | Descrizione |
|---|---|---|
| `"label"` | `str` | Nome visualizzato nel dropdown |
| `"category"` | `str` | `"card"` / `"button"` / `"bet"` / `"player"` / `"pot"` |
| `"get_rect"` | `() → list` | Restituisce la ROI corrente |
| `"set_rect"` | `(rect) → None` | Aggiorna il `_Ref` locale |
| `"read_fn"` | `(img) → str` | Legge la ROI e restituisce feedback testo |
| `"save_fn"` | `() → None` | Scrive in `calibration_data.json` tramite `_save_to_json` |
| `"restore_fn"` | `() → None` | Ripristina il valore baseline |
| `"extra_rects"` | `() → list[list]` | ROI degli altri parametri della stessa categoria |

### Factory `_make_xywh_param`

Genera automaticamente i parametri per `[x, y, w, h]` (button, bet, player) con get/set/save/restore/extra_rects pre-cablati, riducendo il boilerplate da 6×3 = 18 definizioni identiche a 3 chiamate alla factory.

## `_save_to_json(key, val)`

Scrive il valore calibrato in `app/calibration_data.json`.

```python
def _save_to_json(key: str, val) -> None:
    existing = {}
    if _CALIB_PATH.exists():
        with open(_CALIB_PATH, encoding="utf-8") as f:
            existing = json.load(f)
    existing[key] = val
    with open(_CALIB_PATH, "w", encoding="utf-8") as f:
        json.dump(existing, f, indent=2)
```

- **Non modifica `settings.py`**: i valori originali rimangono intatti nel sorgente.
- **Override all'avvio**: il blocco finale di `settings.py` legge `calibration_data.json` (se esiste) e sovrascrive le variabili globali corrispondenti.
- **Accumulo**: ogni chiamata aggiunge/aggiorna solo la chiave specificata — le altre chiavi precedentemente salvate restano intatte.
- `_CALIB_PATH = Path(__file__).parent / "calibration_data.json"`
